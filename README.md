# 🎧 신곡 취향저격 음악 추천 서비스

## 1. 프로젝트 개요
**최근 발매된 음악들** 중에서 사용자의 취향에 맞는 곡을 자동으로 발굴하고 추천하는 웹 서비스입니다.  
실시간 음악 차트 데이터를 기반으로, 사용자가 선호하는 **장르 및 분위기(Emotion)** 를 선택하면 그에 맞는 신곡 목록을 추천받을 수 있습니다.  

---

## 2. 주요 목표
- 실시간으로 최신 곡 데이터를 수집하고 저장
- 음악의 메타데이터(장르, 아티스트, 순위 등)와 오디오 분석 기반 감정 정보를 결합
- 사용자의 취향(장르 + 분위기)에 따라 곡을 추천하는 인터페이스 제공

---

## 3. 시스템 구성도

**데이터 수집 파이프라인**
```
벅스 실시간 차트 → Kafka → Flink → PostgreSQL
```

**웹 서비스 구성**
```
Frontend(Vue.js) ↔ Backend(Django REST API) ↔ DB(PostgreSQL)
```

---

## 4. 세부 구성

### ① 데이터 수집
- **데이터 출처**: 벅스 실시간 차트
- **수집 항목**
  - 노래 제목, 가수명, 순위, 장르
  - +음악 파일 URL 또는 샘플 오디오, 가사!
- **데이터 파이프라인**
  - Kafka → Flink → PostgreSQL
  - Flink에서 데이터 스트리밍 처리 및 정제
- **추가 분석**
  - Hugging Face의 `emotion-recognition` 모델을 이용해 오디오 감정(잔잔함, 신남, 밝음 등) 추출

---

### ② 백엔드 (Django)
- **주요 기능**
  - DB 연동 및 REST API 구축
  - 음악 정보 API 제공 (노래 목록, 상세, 좋아요, 사용자 관리 등)
  - 사용자별 추천곡 리스트 생성 로직 포함
- **API 예시**
  - `/songs/` : 전체 신곡 목록 조회
  - `/songs/{id}/` : 신곡 상세 정보 조회
  - `/recommend/` : 사용자의 취향 기반 추천 재생목록
  - `/user/likes/` : 좋아요 기록 관리

---

### ③ 프론트엔드 (Vue.js)
- **구현 내용**
  - 백엔드 API 연동
  - 사용자가 장르/분위기를 선택할 수 있는 UI 구현
  - 추천 결과(재생목록) 렌더링
  - 로그인/회원가입/좋아요/로그아웃 기능 구현
- **라우팅 구성**
  - `/` : 메인(검색/추천 인터페이스)
  - `/playlist` : 추천곡 재생목록 페이지
  - `/mypage` : 내 활동(좋아요/청취 내역)

---

### ④ 활동 기록 및 대시보드
- **DB 저장 항목**
  - 좋아요, 재생, 검색 등 사용자 활동 로그
- **시각화**
  - Vue 차트 라이브러리를 이용해 사용자 취향 통계 시각화
  - 예: 선호 장르 비율, 감정별 청취 빈도, 최근 재생 순위 변화

---

## 5. 기술 스택

| 구분 | 기술 |
|------|------|
| 데이터 수집 | Kafka, Flink, PostgreSQL, BeautifulSoup |
| 백엔드 | Django REST Framework |
| 프론트엔드 | Vue.js, Axios, Chart.js |
| AI 분석 | Hugging Face `emotion-recognition` 모델 |
| 배포(선택) | Docker, Nginx |

---

## 6. 기대 효과
- 사용자는 단순 차트 정보가 아닌 **취향 기반 신곡 추천 경험**을 얻을 수 있음
- 데이터 기반의 **음악 감정 분석 및 개인화 추천 기술**을 실습 가능
- 팀은 **Kafka–Flink–Django–Vue 파이프라인 전주기 구축 경험**을 습득할 수 있음
